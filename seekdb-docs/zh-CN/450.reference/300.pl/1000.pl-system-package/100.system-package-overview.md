# 系统包概述

seekdb 支持 PL 系统包，以扩展数据库功能并提供对 SQL 功能的 PL 访问。

您可以在创建应用程序时或者在创建自己的存储过程时使用所提供的系统包。本手册介绍了 seekdb 所支持的系统包。

系统包是存储在数据库中的相关程序对象的封装集合。程序对象包括存储过程、函数，变量、常量、游标和异常。

当具有相应权限后，在 `oceanbase` Schema 下运行如下语句查询当前支持的系统包及其相关信息：

```sql
USE oceanbase;
SELECT * FROM __all_package;
```

## PL 系统包的构成

PL 系统包有两个部分：包头（`SPECIFICATION`）和包主体（`BODY`）。包头是应用程序的接口，包含对应用程序可见的公共声明，声明了可使用的类型、变量、常量、异常、游标和子程序。包主体包含实现细节和对应用程序隐藏的私有声明，完整定义了游标和子程序，以支持包头的实现。

  :::info

  seekdb 不支持用户修改系统包。

  :::

## PL 系统包的使用

创建数据库时，将自动安装 seekdb 所提供的系统包。

## PL 系统包概览

下表列出了 seekdb 所支持的 PL 系统包和简要描述。

|                      系统包                      |            描述                    |
|-------------------------------------------------|------------------------------------|
| [DBMS_RESOURCE_MANAGER](13300.dbms-resource-manager/100.dbms-resource-manager-overview.md)| 用于维护资源使用计划，资源使用组和资源计划指令。|
| [DBMS_STATS](15900.dbms-stats/100.dbms-stats-overview.md)        | 查看和修改优化器为数据库对象收集的统计信息。                        |
| [DBMS_UDR](17800.dbms-udr/100.dbms-udr-overview.md)| 提供改写绑定功能，可以在 SQL 开始执行之前根据匹配规则改写数据库接收到的 SQL 语句。|
| [DBMS_VECTOR](17850.dbms-vector/100.dbms-vector-overview.md)|用于管理向量索引。|
| [DBMS_WORKLOAD_REPOSITORY](17900.dbms-workload-repository/100.dbms-workload-repository-overview.md)|  用于管理自动工作负载存储库 (WR)。 |
| [DBMS_MVIEW](9950.dbms-mview/100.dbms-mview-overview.md)  | 使调用者可以了解物化视图和潜在物化视图的功能，包括它们的查询改写可用性。还可以刷新不属于同一刷新组的物化视图并清除日志。    |
| [DBMS_MVIEW_STAT](10050.dbms-mview-stat/100.dbms-mview-stat-overview.md)  | 使调用者可以使用接口来管理物化视图刷新操作的统计信息的收集和保留。    |
| [DBMS_PARTITION](13200.dbms_partition/100.dbms_partition-overview.md)| 用于提供动态分区管理接口。|
