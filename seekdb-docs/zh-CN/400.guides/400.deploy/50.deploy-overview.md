# 部署概述

seekdb 提供两种部署模式，您可以根据业务场景需要选择合适的部署模式。

* 嵌入式模式：seekdb 作为一个“库”运行在你的应用程序内部，它可以轻松跑在各种“端“上，适合移动应用开发，桌面应用开发、物联网（IoT）与边缘设备上的应用开发。该模式下可以通过 Python 脚本或者 SDK 方式连接并进行 seekdb 的管理和使用。

* 服务器模式：采用单机部署，适合 CI/CD、测试，简单且运维成本低，服务器启动快。该模式下通过 MySQL 客户端连接，或者通过 SDK 连接，进行  seekdb 的管理和使用。

## 部署方式

不同的部署模式提供了多种部署方式，可以按照需求进行选择。

* 嵌入式模式：

   * 通过 seekdb 提供的 pyseekdb SDK 部署 seekdb。
 
      详细的部署操作，参见 [嵌入式部署模式](600.python-seekdb.md)。

* 服务器模式：
  
  * 通过 yum install 部署 seekdb。
    
      详细的部署操作，参见 [通过 yum install 部署 seekdb](700.server-mode/100.deploy-by-systemd.md)。
  
  * 部署 seekdb 容器环境。
    
      详细的部署操作，参见 [部署 seekdb 容器环境](700.server-mode/200.deploy-by-docker.md)。
  
  * 部署 OceanBase Desktop。
    
      OceanBase Desktop 是一个用于管理和操作 seekdb 的桌面应用程序。它提供了图形化界面，使用户能够方便地进行数据库管理、查询执行、数据导入导出等操作。

      详细的部署操作，参见 [部署 OceanBase Desktop](700.server-mode/300.deploy-oceanbase-desktop.md)。
