# 部署前配置概述

为了保证 seekdb 能够更好的使用，需要对操作系统和安装环境进行配置，分别如下:

| 配置项 | 个人试用场景 | 生产环境 | 说明 |
| ------ | ------- | ------- | ---- |
| [创建用户](300.create-regular-user.md) | 可以使用 root 用户。 | 建议使用具有 sudo 权限的用户。 | 可以使用带有 sudo 权限的普通用户。 |
| [规划磁盘](400.plan-disks.md) | 无需配置 | 建议配置 | 建议在生产环境上日志保存目录与数据保存目录使用不同的磁盘，避免因两个目录规划在同一块磁盘上在高压力场景下出现性能问题。|
| [配置 limits.conf](500.configure-limits-conf.md) | 无需配置 | 建议配置 | 提高运行时的稳定性和性能。 |
| [配置 sysctl.conf](600.configure-sysctl-conf.md) | 无需配置 | 建议配置 | 提高 Linux 系统的性能，从而提高 seekdb 性能。 |
| [关闭防火墙和 SELinux](700.disable-firewall-and-selinux.md) | 建议配置 | 建议配置 | 防止被拦截而无法访问数据库。 |
