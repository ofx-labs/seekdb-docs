# SET ROLE

## Description

This statement is used to activate roles granted to the current login user in the current session.

:::info
Roles activated by the `SET ROLE` statement only affect the current session and do not affect subsequent sessions.
:::

## Syntax

```sql
SET ROLE {
    DEFAULT
    | NONE
    | ALL
    | ALL EXCEPT role_name [, role_name ...]
    | role_name [, role_name ...]
};
```

## Parameters

|       **Parameter**      | **Description** |
|---------------------|----------|
| DEFAULT             | Specifies to keep the default role setting. The system will activate roles based on the value of the variable `activate_all_roles_on_login` and the default activated roles set by the `SET DEFAULT ROLE` statement or the `ALTER USER` statement:<ul><li>If the value of the variable <code>activate_all_roles_on_login</code> is <code>on</code>, all roles granted to the user are activated in the session.</li><li>If the value of the variable <code>activate_all_roles_on_login</code> is <code>off</code>, the default activated roles specified by the <code>SET DEFAULT ROLE</code> statement or the <code>ALTER USER</code> statement are activated in the session.</li></ul>|
| NONE                | Specifies to disable all roles granted to the user.|
| ALL                 | Specifies to activate all roles granted to the user.|
| ALL EXCEPT          | Specifies to not activate any roles granted to the user.|
| role_name           | Specifies the name of the role. When activating multiple roles, separate the role names with commas (`,`).|

## Examples

* Specify to activate all roles granted to the user except for the `role001` role in the current session.

  ```shell
  SET ROLE ALL EXCEPT role001;
  ```

* Specify to activate the `role001` and `role002` roles of the user in the current session.

  ```shell
  SET ROLE role001, role002;
  ```
