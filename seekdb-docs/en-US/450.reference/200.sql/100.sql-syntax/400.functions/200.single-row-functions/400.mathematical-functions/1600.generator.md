# GENERATOR

## Declaration

```sql
GENERATOR(N)
```

## Description

This function is an inbuilt function in the Table Function framework. It allows you to call it in a Table Function and returns N rows of data. N is a 64-bit unsigned integer greater than or equal to 0.

A table function is a function used in SQL that returns a data table as a result. Unlike traditional SQL functions that return scalar values, table functions can return datasets with multiple rows and columns.

`TABLE(GENERATOR(N))` does not generate any data. It only generates N consecutive rows. If you need to generate data, you must add specific expressions, such as constants, variables, functions, or UDFs, after `SELECT`. `TABLE(GENERATOR(N))` can be used in parallel operations, such as parallel `SELECT` and parallel DML. However, only one thread is responsible for reading the rows generated by `TABLE(GENERATOR(N))`, which means that the DFO containing `TABLE(GENERATOR(N))` has a `DOP=1`.
